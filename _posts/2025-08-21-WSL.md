In case if you are wondering, Windows Subsystem for Linux ( [WSL](https://learn.microsoft.com/en-us/windows/wsl/about))  is a feature of Windows that allows you to run a Linux environment on your Windows machine.

## Basic commands for WSL
First make sure, the WSL and Virtual Machine Platform features are enabled.

![Windows Feature](/assets/img/post/wsl/windows-feature.png)

```bash
wsl --install # Install WSL with default Ubuntu distro
wsl --list --online # List available Linux distro
wsl --list --verbose # List installed distro
wsl --set-default <Distribution Name> 
wsl ~ # Start WSL in user's home
wsl --distribution <Distribution Name> --user <User Name> #Run a specific Linux distribution from PowerShell or CMD
wsl --update # Update WSL
wsl --status # Check WSL status
wsl --version # Check WSL version
wsl --help
wsl --user <Username> #Run as a specific user
<DistributionName> config --default-user <Username> #Change the default user for a distribution
wsl --shutdown 
wsl --terminate <Distribution Name> 
```

```bash
# Identify IP Address
wsl hostname -I  # Returns the IP address of your Linux distribution installed via WSL 2 (the WSL 2 VM address)

ip route show | grep -i default | awk '{ print $3}' #Returns the IP address of the Windows machine as seen from WSL 2 (the WSL 2 VM)
```

```bash
wsl --export <Distribution Name> <FileName> # Export a distribution
wsl --import <Distribution Name> <InstallLocation> <FileName> # Import a distribution
wsl --import-in-place <Distribution Name> <FileName> # Import a distribution in place
wsl --unregister <DistributionName> # Unregister or uninstall a Linux distribution
wsl --mount <DiskPath> # Mount a disk or device
wsl --unmount <DiskPath> # Unmount disks
```

### From Windows command prompt
```bash
code --remote wsl+<distro name> <path in WSL> 
# code --remote wsl+Ubuntu /home/satish/projects
```
